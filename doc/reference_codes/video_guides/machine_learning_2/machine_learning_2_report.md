## TwinCAT And Deployment Implications  

1. **Export from TwinCAT → ML orchestration**  
   - The `P_Experiment_Cam_with_TECompens` block is exported as a TEFF file (`TEFF_File = P_Experiment_Cam_wi`). This must be imported into the PLC (Beckhoff) to keep the same execution order.  

2. **PLC‑side ML orchestration**  
   - The `ae targetVelocity_TE 4` line is generated by TwinCAT’s prediction block and is interpreted by the PLC as a *target velocity* command. Any change in this value must be reflected in the Beckhoff test rig’s motion controller.  

3. **TestRig structure**  
   - Fast tasks (`Task Fast Task Fast (250s) (2501s)`) are linked to `FB_BoschMotor` and `FB_LightDevice`. The “low correction” flag from the 500 µs timing issue disables certain transmissions, which is why the ML block only activates after a steady state.  

4. **Code‑adaptation considerations**  
   - Ensure the PLC’s CSV reader (`needToReadCSV_`) matches the TwinCAT `Db xCSV_and_Vectors D>` address (0x6F4B9).  
   - The Boolean filter (`xFiltering Type BOOL`) must be set to *true* only when the 500 µs task has completed; otherwise the ML block will fire prematurely.
